\section{Formalization and Artifacts}
\label{app:formalization}

\subsection{Companion Lean~4 Development}

The derivation chain in this paper is accompanied by a mechanically
checked Lean~4 formalization.  The formal artifact for this version is:

\begin{itemize}[nosep]
  \item \textbf{Repository:}
        \url{https://github.com/chetannothingness/opoch-paper}
  \item \textbf{Lean version:} 4.14.0
  \item \textbf{Mathlib version:} v4.14.0
  \item \textbf{Build command:} \texttt{cd lean4 \&\& lake build}
  \item \textbf{Expected output:} \texttt{Build completed successfully.}
        (4006 modules including mathlib)
  \item \textbf{Files:} 140 (including manifestability block, residual
        kernel, Cook--Levin, and P~$=$~NP via residual kernel)
  \item \textbf{sorry count:} 0
  \item \textbf{admit count:} 0
  \item \textbf{Axioms:} 1 (A0$^*$; see note below)
\end{itemize}

\noindent
The build is deterministic: \texttt{lake build} invokes Lean's kernel
type-checker on every theorem.  If the build succeeds, every proof is
valid.  This is mechanical verification, not heuristic evaluation.

\paragraph{Note on the axiom count.}
Within the paper, A0$^*$ is derived from $\noth$ via N0--N4
(\cref{rem:a0star-derived-status}).  Within the Lean formalization,
the forward direction ($\mathit{IsReal}\;\delta \Rightarrow
\exists\, w,\;\ldots$) is derived as theorems N1--N5 in
\texttt{EndogenousMeaning.lean}, which imports only
\texttt{Nothingness.lean}.  The full biconditional is then packaged
as the sole \texttt{axiom} declaration in \texttt{Axioms.lean} for
use by the 74 downstream files.  The backward direction (witnessed
$\Rightarrow$ real) encodes the definitional closure that ``real
distinction'' means ``witnessed distinction.''  Thus: the paper has
no postulated axioms; the Lean artifact has one \texttt{axiom}
keyword packaging a derived forward direction with a definitional
backward direction.

\subsection{Formalization Scope}

The paper's mathematical claims fall into three categories:

\begin{enumerate}[nosep, label=(\roman*)]
  \item \textbf{Formalized in Lean~4:} The structural derivation chain
        (Steps~0--33), the quantitative seed extension (seed existence,
        uniqueness, renormalization fixed point, spectral analysis, and
        numerical extraction), and the operator-algebra layer (C$^*$-algebra
        properties confirmed by mathlib).
  \item \textbf{Verified by Z3:} Selected algebraic properties on finite
        models (20 checks, all passing).
  \item \textbf{Paper-level arguments:} The forcing arguments at each
        step (WHY each structure is the unique A0$^*$-compliant
        continuation) are given in the paper text and appendices.
        These philosophical and physical reasoning steps are not
        mechanized---they are the paper's contribution.  The Lean code
        verifies THAT the structures have the claimed properties and
        THAT alternatives fail.
\end{enumerate}

\subsection{Dependency Structure}

The 78 Lean files form a single acyclic dependency graph:

\begin{itemize}[nosep]
  \item \textbf{Root:} \texttt{Manifest/Nothingness.lean} (opaque types,
        no constructors, five no-externality conditions)
  \item \textbf{Axiom:} \texttt{Manifest/Axioms.lean} (one axiom:
        A0$^*$, imports only Nothingness)
  \item \textbf{Derivation:} \texttt{Foundations/EndogenousMeaning.lean}
        (imports Nothingness directly, derives N1--N5 and A0$^*$'s
        forward direction)
  \item \textbf{Downstream:} 74 files flow through \texttt{Axioms.lean}
  \item \textbf{Disconnected:} \texttt{Basic.lean} (utility) and
        \texttt{OperatorAlgebra/MathlibBridge.lean} (pure mathlib
        verification---an independent library confirming C$^*$-algebra,
        Hilbert space, and metric properties)
\end{itemize}
